Java Lombok Annotations: A Practical Guide

Lombok is a Java library that reduces boilerplate code through compile-time code generation. It provides annotations that generate getters, setters, constructors, builders, and more, making Java code more concise and maintainable.

1) Quick Overview

  • Lombok: compile-time code generation via annotations; no runtime overhead.
  • Setup: Add dependency (Maven/Gradle) and enable annotation processing in your IDE.
  • Common annotations: @Builder, @Getter/@Setter, @Data, @NoArgsConstructor, @AllArgsConstructor, @RequiredArgsConstructor, @ToString, @EqualsAndHashCode, @Slf4j, @Value.
  • Best practice: Use Lombok judiciously; understand what code it generates.

2) Constructor Annotations

2.1 @NoArgsConstructor

Generates a no-argument constructor. Useful for JPA entities, frameworks requiring default constructors.

@NoArgsConstructor
public class User {
    private Long id;
    private String name;
    private String email;
}

// Generated:
// public User() {}

2.2 @AllArgsConstructor

Generates a constructor with all fields as parameters.

@AllArgsConstructor
public class Point {
    private int x;
    private int y;
}

// Generated:
// public Point(int x, int y) { this.x = x; this.y = y; }

2.3 @RequiredArgsConstructor

Generates a constructor for final fields and fields marked with @NonNull.

@RequiredArgsConstructor
public class Service {
    private final UserRepository repository;
    private final EmailService emailService;
    private String optionalField;
}

// Generated:
// public Service(UserRepository repository, EmailService emailService) {
//     this.repository = repository;
//     this.emailService = emailService;
// }

3) Builder Pattern

3.1 @Builder

Generates a fluent builder API. Excellent for objects with many optional fields.

@Builder
@Getter
public class Product {
    private Long id;
    private String name;
    private String description;
    @Builder.Default
    private BigDecimal price = BigDecimal.ZERO;
    private Integer stock;
}

// Usage:
Product product = Product.builder()
    .id(1L)
    .name("Laptop")
    .description("High-performance laptop")
    .price(new BigDecimal("1299.99"))
    .stock(50)
    .build();

3.2 @Builder with @Singular

For collections, @Singular generates add methods.

@Builder
@Getter
public class Order {
    private Long id;
    @Singular
    private List<OrderItem> items;
}

// Usage:
Order order = Order.builder()
    .id(1L)
    .item(OrderItem.builder().name("Item1").build())
    .item(OrderItem.builder().name("Item2").build())
    .build();

4) Accessor Annotations

4.1 @Getter and @Setter

Generate getter and setter methods.

@Getter
@Setter
public class Person {
    private String firstName;
    private String lastName;
    @Getter(AccessLevel.PRIVATE)
    private String password; // Only getter is private
}

4.2 @Data

Combines @Getter, @Setter, @ToString, @EqualsAndHashCode, and @RequiredArgsConstructor.

@Data
public class Employee {
    private Long id;
    private String name;
    private String email;
    private final LocalDateTime createdAt;
}

// Generated: getters, setters, toString, equals, hashCode, constructor for final fields

5) Utility Annotations

5.1 @ToString

Generates toString() method. Use @ToString.Exclude to exclude fields.

@ToString
public class User {
    private Long id;
    private String name;
    @ToString.Exclude
    private String password; // Excluded from toString
    @ToString.Include(name = "emailAddress")
    private String email;
}

5.2 @EqualsAndHashCode

Generates equals() and hashCode(). Use @EqualsAndHashCode.Exclude to exclude fields.

@EqualsAndHashCode
public class Product {
    private Long id;
    @EqualsAndHashCode.Exclude
    private LocalDateTime lastModified; // Excluded from equals/hashCode
    private String name;
}

5.3 @Value

Creates immutable classes. All fields are final, generates getters, toString, equals, hashCode, and an all-args constructor.

@Value
@Builder
public class ImmutableUser {
    Long id;
    String name;
    String email;
}

// All fields are final, no setters generated

6) Logging

6.1 @Slf4j

Generates a static log field using SLF4J.

@Slf4j
public class UserService {
    public void createUser(String name) {
        log.info("Creating user: {}", name);
        // ... implementation
    }
}

// Generated:
// private static final Logger log = LoggerFactory.getLogger(UserService.class);

7) Practical Examples

7.1 JPA Entity with Lombok

@Entity
@Table(name = "users")
@Data
@NoArgsConstructor
@AllArgsConstructor
@Builder
public class User {
    @Id
    @GeneratedValue(strategy = GenerationType.IDENTITY)
    private Long id;

    @Column(nullable = false)
    private String name;

    @Column(unique = true)
    private String email;

    @ToString.Exclude
    @EqualsAndHashCode.Exclude
    private String password;
}

7.2 DTO with Builder

@Value
@Builder
public class UserDTO {
    Long id;
    String name;
    String email;
    @Builder.Default
    boolean active = true;
}

// Usage:
UserDTO dto = UserDTO.builder()
    .id(1L)
    .name("John Doe")
    .email("john@example.com")
    .build();

@Slf4j

8) Test Strategies

8.1 Testing Builder Pattern

@Test
void builderCreatesObjectCorrectly() {
    Product product = Product.builder()
        .id(1L)
        .name("Test Product")
        .price(new BigDecimal("99.99"))
        .build();

    assertEquals(1L, product.getId());
    assertEquals("Test Product", product.getName());
    assertEquals(new BigDecimal("99.99"), product.getPrice());
}

8.2 Testing Equals and HashCode

@Test
void equalsAndHashCodeWorkCorrectly() {
    Product p1 = Product.builder().id(1L).name("Product").build();
    Product p2 = Product.builder().id(1L).name("Product").build();

    assertEquals(p1, p2);
    assertEquals(p1.hashCode(), p2.hashCode());
}

8.3 Testing ToString

@Test
void toStringIncludesRelevantFields() {
    User user = User.builder()
        .id(1L)
        .name("John")
        .email("john@example.com")
        .build();

    String str = user.toString();
    assertTrue(str.contains("John"));
    assertTrue(str.contains("john@example.com"));
    assertFalse(str.contains("password")); // Excluded field
}

8.4 Testing Immutable Value Objects

@Test
void valueObjectIsImmutable() {
    ImmutableUser user = ImmutableUser.builder()
        .id(1L)
        .name("John")
        .email("john@example.com")
        .build();

    // No setters available - compile error if attempted
    assertEquals("John", user.getName());
}

9) Best Usage Scenarios

@Builder

  • DTOs with many optional fields.
  • Configuration objects.
  • Test data builders.
  • API request/response objects.

@Data

  • Simple POJOs/DTOs.
  • JPA entities (with caution - be mindful of equals/hashCode).
  • Data transfer objects.

@Value

  • Immutable value objects.
  • DTOs that should not be modified.
  • Domain value objects.

@RequiredArgsConstructor

  • Dependency injection in Spring (alternative to @Autowired).
  • Services, repositories, controllers.
  • Classes with final dependencies.

@Slf4j

  • Any class that needs logging.
  • Services, controllers, utilities.

@NoArgsConstructor / @AllArgsConstructor

  • JPA entities (framework requirements).
  • Framework integration classes.
  • Simple data containers.

10) Pitfalls and Best Practices

Avoid

  • Using @Data on JPA entities without careful consideration of equals/hashCode (can cause issues with lazy loading).
  • Overusing Lombok when explicit code is clearer.
  • Ignoring what Lombok generates (understand the code).

Best Practices

  • Use @EqualsAndHashCode.Exclude for lazy-loaded JPA relationships.
  • Prefer @RequiredArgsConstructor over @Autowired for dependency injection.
  • Use @Builder.Default for default values in builders.
  • Combine @Value with @Builder for immutable objects with builder pattern.
  • Use @ToString.Exclude for sensitive fields (passwords, tokens).

JPA Entity Considerations

@Entity
@Data
@NoArgsConstructor
@AllArgsConstructor
@Builder
public class Order {
    @Id
    @GeneratedValue
    private Long id;

    @ManyToOne(fetch = FetchType.LAZY)
    @EqualsAndHashCode.Exclude
    @ToString.Exclude
    private User user; // Exclude from equals/hashCode to avoid lazy loading issues

    private BigDecimal total;
}

11) IDE Setup

  • IntelliJ IDEA: Install Lombok plugin, enable annotation processing.
  • Eclipse: Install Lombok, restart IDE.
  • VS Code: Install Lombok extension.

12) Maven/Gradle Dependency

<!-- Maven -->
<dependency>
    <groupId>org.projectlombok</groupId>
    <artifactId>lombok</artifactId>
    <version>1.18.30</version>
    <scope>provided</scope>
</dependency>

// Gradle
dependencies {
    compileOnly 'org.projectlombok:lombok:1.18.30'
    annotationProcessor 'org.projectlombok:lombok:1.18.30'
}
